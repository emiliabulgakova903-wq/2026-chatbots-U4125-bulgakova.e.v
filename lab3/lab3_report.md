## 2. Процесс деплоя

### Пошаговая инструкция

**Шаг 1. Подготовка репозитория**
```
lab3/
├── bot.py
├── database.py
├── equipment.py
├── equipment.csv
└── requirements.txt
amvera.yaml          ← в корне репозитория
```

**Шаг 2. Создание `amvera.yaml` в корне репозитория**
```yaml
meta:
  environment: python
  toolchain:
    name: pip
    version: "3.10"
build:
  requirementsPath: lab3/requirements.txt
run:
  scriptName: lab3/bot.py
  persistenceMount: /data
```

**Шаг 3. Создание проекта на Amvera**
1. Зарегистрироваться на [amvera.ru](https://amvera.ru)
2. Создать новый проект → тип **Python**
3. Подключить GitHub-репозиторий
4. В настройках указать `scriptName: lab3/bot.py`

**Шаг 4. Настройка переменных окружения**
В дашборде Amvera → раздел «Переменные окружения»:
```
BOT_TOKEN=<токен от BotFather>
```

**Шаг 5. Деплой**
```bash
git add .
git commit -m "Deploy lab3 bot"
git push origin main
```
Amvera автоматически запускает сборку при каждом push в `main`.

### Проблемы и решения при деплое

#### Проблема 1: Токен не найден при запуске
**Ошибка:** `ValueError: Токен бота не найден`  
**Причина:** На сервере нет файла `.env`, а код читал токен только из него.  
**Решение:**
```python
load_dotenv()  # для локального .env
token = os.environ.get("BOT_TOKEN")  # системные переменные Amvera
```

#### Проблема 2: Неверный scriptName в дашборде
**Ошибка:** `python3: can't open file '/app/python3 lab3/bot.py'`  
**Причина:** В поле scriptName было введено `python3 lab3/bot.py` целиком — Amvera приняла это как имя файла.  
**Решение:** В поле `scriptName` — только путь к скрипту: `lab3/bot.py`.

#### Проблема 3: Конфиг читался из неверного места
**Причина:** Файл `amvera.yaml` лежал в `lab3/`, а Amvera читает только из корня репозитория.  
**Решение:** Перенесён в корень репозитория, старый `lab3/amvera.yml` удалён.

#### Проблема 4: Утечка SQLite-соединений
**Симптом:** Бот отвечал после рестарта, затем переставал реагировать.  
**Причина:** `get_connection().execute(...)` без `with` — соединения не закрывались.  
**Решение:** Все запросы обёрнуты в `with get_connection() as conn:`.

#### Проблема 5: Пути к файлам не работали на сервере
**Причина:** Пути `"equipment.csv"` и `"tasks.db"` искались в рабочей директории (корень), а не рядом со скриптом.  
**Решение:**
```python
_HERE = os.path.dirname(os.path.abspath(__file__))
DB_PATH = "/data/tasks.db" if os.path.exists("/data") else os.path.join(_HERE, "tasks.db")
```

#### Проблема 6 (главная): Дедлок HTTP-соединений
**Симптом:** Бот получал сообщения (`UPDATE RECEIVED` в логах), вызывал обработчик (`cmd_start вызван`), но никогда не отправлял ответ — `sendMessage` отсутствовал в логах.  
**Причина:** Использовался один экземпляр `HTTPXRequest` для `getUpdates` (держит соединение 10 сек.) и для `sendMessage`. Пул блокировался — дедлок.  
**Решение:** Два независимых экземпляра:
```python
request_config = HTTPXRequest(connect_timeout=30.0, read_timeout=30.0, ...)
get_updates_request = HTTPXRequest(connect_timeout=30.0, read_timeout=30.0, ...)
app = ApplicationBuilder()
    .request(request_config)
    .get_updates_request(get_updates_request)
    .build()
```

#### Проблема 7: Лавина старых сообщений при рестарте
**Причина:** Telegram накапливал команды за время простоя; при запуске бот падал на одной из них.  
**Решение:** `app.run_polling(drop_pending_updates=True)` — очередь сбрасывается при старте.

## 5. Улучшения по результатам фидбека

### Что изменили

**Улучшение 1: Подсказка формата даты**

В диалоге `/add` на шаге ввода напоминания добавлен пример:
```
⏰ Укажите дату и время напоминания в формате:
ДД.ММ.ГГГГ ЧЧ:ММ

Пример: 15.04.2026 09:00

Или введите - чтобы пропустить.
```

**Улучшение 2: Инфраструктурные улучшения (по итогам деплоя)**

Добавлен глобальный обработчик ошибок — при любой внутренней ошибке пользователь получает сообщение вместо молчания бота:
```python
async def error_handler(update, context):
    logger.error("Ошибка:", exc_info=context.error)
    if update and update.effective_message:
        await update.effective_message.reply_text(
            "⚠️ Произошла внутренняя ошибка. Попробуйте /start."
        )
app.add_error_handler(error_handler)
```

### Как это помогло

- Пользователи перестали сталкиваться с молчащим ботом при ошибках
- Формат даты стал понятен с первого раза — повторных вопросов не было
