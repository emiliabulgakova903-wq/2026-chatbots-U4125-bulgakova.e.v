# Tracker Production Improvement Service Bot

Telegram-бот для отслеживания задач по улучшению производства.  
Поддерживает приоритеты, напоминания и хранит данные в SQLite.

Демонстрация работы: https://drive.google.com/drive/folders/1W19o3c4mP9oV4cj6vXRedldbd0oN4JTT
---

## Структура проекта

```
tracker_bot/
├── bot.py           # Основной код бота (обработчики команд, диалоги)
├── database.py      # Работа с SQLite: создание таблиц, CRUD-операции
├── tasks.db         # База данных (создаётся автоматически при первом запуске)
├── requirements.txt # Зависимости Python
├── .env             # Файл с токеном бота (создать из .env.example)
├── .env.example     # Пример конфигурации
└── README.md        # Документация
```

## Быстрый старт

### 1. Клонируйте / скачайте проект

```bash
cd tracker_bot
```

### 2. Создайте виртуальное окружение и установите зависимости

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
```

### 3. Создайте файл `.env`

```bash
cp .env.example .env
```

Откройте `.env` и вставьте токен вашего бота:

```
BOT_TOKEN=123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> Токен выдаётся при создании бота через [@BotFather](https://t.me/BotFather) в Telegram.

### 4. Запустите бота

```bash
python bot.py
```

При первом запуске автоматически создаётся файл `tasks.db` с нужными таблицами.

## Команды бота

| Команда   | Описание                                                         |
|-----------|------------------------------------------------------------------|
| `/start`  | Приветствие и краткая инструкция                                 |
| `/add`    | Мастер добавления задачи (название → приоритет → напоминание)    |
| `/tasks`  | Список всех активных задач с кнопками управления                 |
| `/help`   | Подробная справка                                                |
| `/cancel` | Отменить текущий диалог добавления задачи                        |

## Приоритеты задач

| Значок | Приоритет | Описание                  |
|--------|-----------|---------------------------|
| 🔴     | Высокий   | Срочные задачи            |
| 🟡     | Средний   | Плановые задачи           |
| 🟢     | Низкий    | Задачи по возможности     |

В списке `/tasks` задачи отсортированы: сначала Высокий приоритет, затем Средний, затем Низкий.

---

## Напоминания

При добавлении задачи через `/add` можно указать дату и время напоминания.

**Формат:** `ДД.ММ.ГГГГ ЧЧ:ММ`  
**Пример:** `15.04.2026 09:00`

Чтобы пропустить напоминание, введите `-` или `нет`.

Напоминания восстанавливаются из базы данных при каждом перезапуске бота — задачи не пропадут.

## База данных

Используется **SQLite** (файл `tasks.db`). Схема таблицы `tasks`:

| Поле         | Тип     | Описание                             |
|--------------|---------|--------------------------------------|
| `id`         | INTEGER | Первичный ключ (автоинкремент)        |
| `user_id`    | INTEGER | Telegram ID пользователя             |
| `title`      | TEXT    | Текст задачи                         |
| `priority`   | TEXT    | Приоритет: Низкий / Средний / Высокий|
| `remind_at`  | TEXT    | Дата напоминания (`YYYY-MM-DD HH:MM`)|
| `completed`  | INTEGER | 0 — активна, 1 — выполнена           |
| `created_at` | TEXT    | Дата создания                        |
