Разработать Telegram-бота для отслеживания задач джля службы совершенствования производства с использованием библиотеки `python-telegram-bot` версии 20+, локальной базы данных SQLite и системы напоминаний.

Создать бота со следующим функционалом:

## 5. Описание реализации

### 5.1 База данных (`database.py`)

Используется одна таблица `tasks` со следующей схемой:

```sql
CREATE TABLE tasks (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    INTEGER NOT NULL,
    title      TEXT    NOT NULL,
    priority   TEXT    NOT NULL DEFAULT 'Средний',
    remind_at  TEXT,
    completed  INTEGER NOT NULL DEFAULT 0,
    created_at TEXT    NOT NULL
)
```

Реализованы функции: `init_db`, `add_task`, `get_tasks`, `get_task_by_id`, `complete_task`, `delete_task`, `get_completed_tasks`, `get_pending_reminders`.

### 5.2 Команды бота (`bot.py`)

| Команда | Описание |
|---|---|
| `/start` | Приветствие и краткая инструкция |
| `/add` | Мастер добавления задачи (ConversationHandler: название → приоритет → напоминание) |
| `/tasks` | Список активных задач с кнопками «Выполнено» и «Удалить» |
| `/report` | Отчёт по выполненным задачам за выбранный период |
| `/help` | Подробная справка |
| `/cancel` | Отмена текущего диалога |

### 5.3 Приоритеты задач

Задачи сортируются по приоритету:
- 🔴 **Высокий** — срочные задачи
- 🟡 **Средний** — плановые задачи
- 🟢 **Низкий** — задачи по возможности

### 5.4 Система напоминаний

Напоминания реализованы через `job_queue` (APScheduler, встроенный в PTB). Для корректной работы в любом часовом поясе задержка передаётся в секундах:

```python
delay = max(1.0, (remind_at - datetime.now()).total_seconds())
app.job_queue.run_once(callback=send_reminder, when=delay, ...)
```

После перезапуска бота незасработавшие напоминания восстанавливаются из базы данных.

### 5.5 Отчёт (`/report`)

Поддерживаемые периоды: 7 дней, 30 дней, текущий месяц, всё время. При количестве задач ≤ 10 — текстовое сообщение, при > 10 — CSV-файл с кодировкой UTF-8 BOM (корректно открывается в Excel).

### 5.6 Конфигурация и безопасность

Токен бота хранится в файле `.env` и загружается через `python-dotenv`. Файл `.gitignore` исключает `.env` и `*.db` из репозитория.

## 7. Примеры работы бота

### Добавление задачи (`/add`)
1. Пользователь вводит `/add`
2. Бот запрашивает название задачи
3. Пользователь вводит название, бот предлагает выбрать приоритет через Inline-кнопки
4. После выбора приоритета бот запрашивает дату напоминания (формат `ДД.ММ.ГГГГ ЧЧ:ММ`) или «-» для пропуска
5. Задача сохраняется в базе данных

### Список задач (`/tasks`)
Отображаются все активные задачи, отсортированные по приоритету. У каждой задачи есть кнопки:
- ✅ **Выполнено** — помечает задачу выполненной
- 🗑 **Удалить** — удаляет задачу

### Отчёт (`/report`)
Пользователь выбирает период через Inline-кнопки. Бот выводит список выполненных задач в формате:
```
📅 Дата | ✅ Задача | 📈 Приоритет
```
